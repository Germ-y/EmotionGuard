# EmotionGuard 판단 기준

이 문서는 EmotionGuard의 현재 구현 기준을 설명한다. EmotionGuard는 상담을 대신하는 챗봇이 아니라, 고객 음성이 상담사에게 전달되기 전 위험 발화를 감지하고 보호 마스크, 경고, 보고서 기록을 결정하는 중간 레이어다.

## 1. 발화 분류 체계

고객 발화는 정책 엔진에서 아래 상태 중 하나로 정리된다.

| 상태 | 기준 | 주요 조치 |
| --- | --- | --- |
| `normal` | 욕설, 성희롱, 고성이 없는 일반 발화 | 원문 전달 |
| `raised` | RMS 레벨이 기준치를 초과해 고성으로 판단된 발화 | 피치/볼륨 완화, 경고, 단계 상승 |
| `abuse` | 비윤리 표현 사전 또는 맥락 엔진에서 폭언으로 판단된 발화 | 욕설 구간 삐 처리, 경고, 단계 상승, 보고서 |
| `abuse-raised` | 욕설과 고성이 동시에 발생한 발화 | 삐 처리 + 피치/볼륨 완화, 경고, 단계 상승, 보고서 |
| `sexual` | 성희롱 또는 상담사 개인을 겨냥한 성적/스토킹성 발화 | 경고 TTS, 성희롱 단계 상승, 보고서 |

성희롱은 폭언/고성 단계와 별도 카운터로 관리하며, 화면에서는 성희롱 2단계로 표시한다.

## 2. 즉시 보호 판단

즉시 보호 경로는 `analysisMode=immediate`로 동작한다.

- Web Speech API의 STT 인터림 또는 최종 텍스트를 백엔드로 보낸다.
- 백엔드는 비윤리/성희롱 표현 사전을 먼저 검사한다.
- 욕설이 감지되면 timestamp 추정값을 기반으로 해당 단어 구간에 비프음을 적용한다.
- 단어만으로 판단하기 어려운 욕설 후보는 즉시 삐 처리하지 않고 맥락 판단으로 넘긴다. 예: `고양이 새끼`, `새끼 고양이`는 정상 문맥이 가능하므로 LLM/fallback 맥락 확인 후 처리한다.
- 성희롱 사전에 걸리면 즉시 성희롱 이벤트로 기록하고, 명시적 사전 표현은 해당 구간 삐 처리와 경고/보고서 액션을 만든다.
- 고성은 프론트엔드 RMS 분석으로 판단하며, 42% 기준을 250ms 이상 넘으면 `raised`로 본다.

현재 사전 크기:

- 욕설 사전: 223개
- 욕설 예외어: 7개
- 성희롱 사전: 391개
- 성희롱 예외어: 90개
- fallback 고위험 단어: 10개

`backend/app/data/dictionaries.aihub.json`이 존재하면 AI-Hub 텍스트 윤리검증 데이터에서 추출한 후보 사전을 추가로 병합한다. 이 확장 사전은 git에 커밋하지 않는 로컬 생성물이며, 전체 발화가 아니라 `mapped_slots[].token` 어휘 단위만 사용한다. 자세한 생성 절차는 [AI-Hub 사전 확장 파이프라인](aihub-dictionary.md)을 참고한다.

예외어는 `시발점`, `시발역`처럼 욕설 문자열을 포함하지만 민원 문맥에서 정상 의미가 가능한 단어를 오탐하지 않기 위해 먼저 제외한다.

## 3. 고성 판단

고성은 브라우저의 Web Audio API 분석값으로 처리한다.

- 분석 단위: requestAnimationFrame 기반 실시간 RMS
- 기본 기준값: 42%
- 조절 범위: 20% ~ 80%
- 지속 조건: 기준값 초과 250ms 이상
- 완화 방식: Jungle Pitch Shifter + 출력 게인 조절
- 성별 보정: 남성 base 0.10, 여성 base 0.07

순간적인 박수, 잡음, 마이크 튐을 줄이기 위해 250ms 지속 조건을 둔다.

## 4. 3초 맥락 판단

맥락 경로는 `analysisMode=context_snapshot`으로 동작한다.

1. 프론트엔드가 최근 STT 텍스트를 3초 단위 스냅샷으로 구성한다.
2. 프론트엔드가 RMS, pitch, peak, zero crossing rate, spectral centroid, 발화 속도 추정값을 `audioFeatures`로 함께 보낸다.
3. 백엔드는 먼저 비윤리 표현 사전을 검사한다.
4. 비윤리 표현 사전에 걸리지 않으면 `classify_context()`를 호출한다.
5. `OPENAI_API_KEY`가 있으면 GPT API를 호출한다.
6. OpenAI 키가 비어 있거나 API 오류가 나면 `fallback` 보수 판단으로 돌아간다.

음향 메타데이터는 감정 상태와 긴장도 판단의 보조 근거다. 높은 피치나 큰 음량만으로 성희롱이라고 판단하지 않으며, 성희롱 여부는 발화 내용과 상담사 개인을 겨냥한 맥락을 우선한다.

중요: 현재 로컬 `.env`의 `OPENAI_API_KEY`가 비어 있으면 실제 LLM은 실행되지 않는다. 이때 응답의 `source`는 `fallback`이며, 화면에는 기본 문맥 엔진으로 표시된다. fallback은 LLM이 아니라 제한된 규칙 기반 보수 판단이다.

fallback은 발표 및 키 없는 개발 환경에서 최소 맥락 예시를 처리하기 위해 다음 조합만 보강한다.

- `퇴근 몇 시` + `기다릴게/찾아갈게` -> 성희롱·스토킹성 위험
- `집/주소/사는 곳` + `알아낼/찾아갈/기다릴` -> 성희롱·스토킹성 위험
- `목소리` + `또 전화/듣고 싶다/반했다/섹시` -> 성희롱 위험
- `개인 연락처/개인 번호/카톡 아이디/인스타` + `알려줘/줘/궁금` -> 성희롱 위험
- `얼굴` + `보여줘/보고 싶다/궁금` -> 성희롱 위험
- `찾아간다/가만 안 둬/죽인다` 등 직접 위협 -> 위협 위험

GPT API가 활성화되면 다음 JSON 기준으로만 응답하도록 프롬프트한다.

```json
{
  "abusive": true,
  "severity": "none | mild | severe",
  "categories": ["욕설", "위협", "성희롱"],
  "emotion": "normal | frustrated | angry | threatening",
  "sexual": false
}
```

## 5. 에스컬레이션 기준

폭언/고성은 4단계, 성희롱은 2단계로 분리한다.

| 구분 | 단계 | 조건 | 자동 안내 |
| --- | --- | --- | --- |
| 폭언/고성 | 1단계 | 첫 위험 이벤트 | 폭언 중단 요청 |
| 폭언/고성 | 2단계 | 반복 위험 이벤트 | 차분한 응대 요청 |
| 폭언/고성 | 3단계 | 반복 폭언/고성 | 법적 조치 가능성 안내 |
| 폭언/고성 | 4단계 | 4회 이상 | 상담 종료 안내 |
| 성희롱 | 1단계 | 첫 성희롱 이벤트 | 성희롱 가능성 경고 |
| 성희롱 | 2단계 | 반복 성희롱 이벤트 | 중단 요구 및 후속 조치 안내 |

중복 단계 상승을 줄이기 위해 동일 발화는 짧은 시간 버킷 안에서 중복 처리하지 않는다.

## 6. 오탐 방지

- TTS 안내 중에는 `announcing` 플래그를 켜서 시스템 자신의 경고 발화를 STT가 다시 감지하지 않게 한다.
- 비윤리 표현 사전은 예외어를 먼저 제거한 뒤 욕설/성희롱 단어를 검사한다.
- 맥락 엔진은 행정, 교통, 수도, 복지, 환경 등 정상 민원 문맥을 우선 고려한다.
- 상담사 개인을 직접 겨냥하는 발화가 명확할 때 성희롱으로 본다.

## 7. 기록과 화면 노출

- 화면에는 원문 문장을 직접 노출하지 않고 `[00:17] 원문 문장 비공개 기록됨`처럼 표시한다.
- 내부 로그와 보고서 복사본에는 증빙을 위해 원문을 보존한다.
- 상담 종료 또는 데모 종료 시 누적 로그 기반 특이민원 보고서를 생성한다.
